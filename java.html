What is java :

Java is a high level language, multithereaded, platform independent language which follow the concept of object oriented programming. Java was develoved by James Gosling in the year 1990 in Sun Microsystem.

Terms/components used in programming language

01. Source code - developer written program, it is written according to programming language syntax.
02. Compiler code - compiler generated program that is converted from source cod.
03. Compiler - it is an translation of program that converts source code into machine language at once.
04. Interpreter - it is also translate program that converts source code into machine language by line by line.
05. Extenaible code - operating system understandable readly executed program .exe file.
06. Compilation - it is a process of translating source code into compiler code.
07. Execution - it is process of running compiled code to get output.

JVM, JDK & JRE

JVM (java vertual machine) - it is the software in the form of interprater written in c language , through which we can execute our java program.

  • JVM is resonsible for converting byte code into machine specific code.

JRE (java runtime envoirment) - it provides an envoirment to execute the java programm. It internally contains JVM which is responsibe to execute a java programm.

JDK (java development kit) - it internally contains JVM & JRE where JVM provides all tools to work with java language.

  • JDK contains both compiler and interprater.

Access Modifiers in java

In java Access Modifiers is of 4 types -

01. Public -

    The public keyword is an access modifier, meaning that it is used to set the access level for classes, attributes, methods and constructors.

// save A.java
package pack;
public class A{
public void msg(){System.out.println("Hello");}
}

// save B.java
package mypack;
import pack.*;
class B{
public static void main(String args[]){
A obj = new A();
obj.msg();
}
}

02. Private -

    The class members which have private keyward in its creation statement, can be only accessed in the same class outside of the class it leads runtime exception.

eg.

public class A{
private int z=10;
public static void main(String[] args){
A r1=new A();
System.out.println(r1.z);
}
}

Output : 10

public class B extends A{
public static void main(String[] args){
A r2=new A();
}
}

Output : Runtime Error

03. Protected -

    A class members which is having protected keyward in its creation statement can be accessiable outside the all class within the same package but outside the package in sub class that to only by using sub class object.
    • This rule is only applicabe for non-static protected members, static protected members can be accessiable by using same class name or by using sub class name.

04. Default -

    Whenever we are not declearing our variable with public, private or protected then bydefault accessiblity modifier will be default.
    eg. - default int a;
    • If accessiblity modifier is default then members will be not accessiable outside the package.

Variables

A variable is a container which holds the value while the Java program is executed. A variable is assigned with a data type.
A variable is the name of a reserved area allocated in memory. In other words, it is a name of the memory location. It is a combination of "vary + able" which means its value can be changed.

In java variables is of three types :

1. Local variable -

    A variable declared inside the body of the method is called local variable. You can use this variable only within that method and the other methods in the class aren't even aware that the variable exists. A local variable cannot be defined with "static" keyword.
2. Instance/global Variable -
    A variable declared inside the class but outside the body of the method, is called an instance variable. It is not declared as static. It is called an instance variable because its value is instance-specific and is not shared among instances.
3. Static variable -
    A variable that is declared as static is called a static variable. It cannot be local. You can create a single copy of the static variable and share it among all the instances of the class. Memory allocation for static variables happens only once when the class is loaded in the memory.
